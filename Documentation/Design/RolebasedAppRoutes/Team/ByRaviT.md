# Mentor expectations from team

After discussing the **platform architecture** with students and mentors, we have prepared an **initial draft of application routes and REST API endpoints** for the following stakeholders:

• Admin
• Student
• SME (Subject Matter Expert)
• Mentor
• Employer

These routes define how each role will **navigate the system and interact with backend services through REST APIs**.

This structure is only a **starting point**.

Software architecture becomes stronger when **multiple minds review it**.

Students, developers, and mentors are invited to:

• Suggest improvements
• Propose additional routes
• Recommend better API structures
• Identify missing use cases
• Simplify navigation where possible

The goal is to design a **clean, scalable, and developer-friendly architecture** for the platform.

Your **changes, extensions, and modifications** are most welcome.

Learning software engineering is not only about writing code —
it is also about **thinking together and designing better systems**.

Let's build this architecture **collaboratively**. 🚀

— Mentor Team
