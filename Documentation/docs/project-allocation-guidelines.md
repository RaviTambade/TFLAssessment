# 🎓 TFL CoMentor – Student Project Allocation Guide

We don’t want students randomly building CRUD apps.
We want **structured, skill-mapped, growth-oriented project allocation**.

# 🧠 Core Philosophy

Project allocation should be based on:

1. Skill Level
2. Concept Coverage
3. Role Alignment
4. Growth Direction
5. Performance Data

This connects directly with TFLCoMentor:

* Skill Taxonomy Engine
* Evaluation System
* Insight Core
* Growth Engine

# 🧩 2️⃣ Student Categorization Framework

Instead of random assignment, classify students:

| Level | Profile      | Project Type            |
| ----- | ------------ | ----------------------- |
| L1    | Beginner     | Guided CRUD + DB        |
| L2    | Foundation   | API + DB + Auth         |
| L3    | Intermediate | Microservice module     |

# 📊 3️⃣ Skill-Based Allocation Matrix

Map projects to your taxonomy:

| Subject  | Concepts                   | Project Type          |
| -------- | -------------------------- | --------------------- |
| OOP      | Encapsulation, Abstraction | Library System        |
| DBMS     | Joins, Indexing            | Inventory Engine      |

This ensures every project strengthens **specific concepts**, not just “features”.

# 🧠 4️⃣ Allocation Decision Algorithm (Mentor Model)

```ascii
IF Coverage < 50% → Assign Guided Project
IF Coverage 50–70% → Assign Moderate Challenge
IF Coverage 70–85% → Assign Independent Module
IF Coverage > 85% → Assign Architecture-Level Project
```

Later, your Growth Engine (Python AI) can automate this.

# 🧠 9️⃣ Mentor Intervention Strategy

You don’t solve their problem.

Instead:

| Situation         | Mentor Action            |
| ----------------- | ------------------------ |
| Student stuck     | Ask debugging questions  |
| Design confusion  | Ask them to draw diagram |
| Performance issue | Ask for logs             |
| Low confidence    | Reduce scope             |

You guide, not rescue.
