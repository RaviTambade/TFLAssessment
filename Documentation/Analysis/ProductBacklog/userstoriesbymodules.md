# TFLCoMentor User Stories Organized by Core Modules

**User stories** according to the **TFLCoMentor module flow**

**Users → Skill System → Assessment → Evaluation → Insights → Growth → Hiring**

we align every story with the **natural lifecycle of a learner in the platform**. This makes the product vision clearer and easier for developers, mentors, and stakeholders to understand.

Below is a **structured reorganization of user stories by module**.

# 1️⃣ Users Module (Membership)

### Goal

Manage identity, access, and profiles of all platform participants.

### User Stories

**Student**

* As a student, I want to register on the platform so that I can access learning and assessments.
* As a student, I want to create and update my profile so that mentors and employers can understand my background.

**Mentor**

* As a mentor, I want to register and define my expertise so that I can guide students.

**SME**

* As an SME, I want to contribute learning and evaluation content for specific technologies.

**Employer**

* As an employer, I want to create a company account so that I can hire skilled candidates.

**Admin**

* As an admin, I want to manage users and roles so that platform access remains secure.


# 2️⃣ Skill System (Skill Taxonomy Module)

### Goal

Define the **knowledge structure** of technologies and skills.

### User Stories

**Admin / SME**

* As an SME, I want to define domains like Software Engineering, Data Engineering, AI etc.
* As an SME, I want to define technologies such as Java, .NET, Python.
* As an SME, I want to define skills under technologies.

**Mentor**

* As a mentor, I want to create learning paths so that students can progress systematically.

**Student**

* As a student, I want to explore skills and learning paths so that I know what to learn next.


# 3️⃣ Assessment Module (Assessment Orchestrator)

### Goal

Conduct structured assessments.

### User Stories

**Mentor**

* As a mentor, I want to schedule assessments for students.

**Student**

* As a student, I want to take assessments to evaluate my knowledge.

**Admin**

* As an admin, I want to assign tests to groups of students.

**System**

* The system should manage assessment attempts and session tracking.


# 4️⃣ Evaluation Module (Evaluation Engine)

### Goal

Automatically evaluate student performance.

### User Stories

**System**

* The system should automatically evaluate MCQ tests.
* The system should execute coding problems in a secure environment.
* The system should evaluate assignments.

**Student**

* As a student, I want to see my test score after submission.

**Mentor**

* As a mentor, I want to review student submissions and scores.

# 5️⃣ Insights Module (Insight Core)

### Goal

Generate meaningful analytics from evaluation data.

### User Stories

**Student**

* As a student, I want to see my skill scores so that I know my strengths.

**Mentor**

* As a mentor, I want to see student skill gaps so that I can guide them.

**Admin**

* As an admin, I want to view platform analytics to track learning effectiveness.

**System**

* The system should calculate concept mastery.
* The system should generate skill gap reports.


# 6️⃣ Growth Module (Growth Engine)

### Goal

Help students improve based on insights.

### User Stories

**Mentor**

* As a mentor, I want to recommend learning activities.

**System**

* The system should suggest learning paths based on skill gaps.

**Student**

* As a student, I want to receive project assignments to improve my skills.

**Community**

* As a student, I want to participate in discussions to learn from peers.


# 7️⃣ Hiring Module (Hiring Engine)

### Goal

Connect skilled candidates with employers.

### User Stories

**Employer**

* As an employer, I want to post job openings.

**Employer**

* As an employer, I want to find candidates based on skill scores.

**Student**

* As a student, I want to apply for jobs relevant to my skills.

**Employer**

* As an employer, I want to schedule interviews with selected candidates.

**System**

* The system should match candidate skills with job requirements.


# Final Product Flow of TFLCoMentor

```
Users
   ↓
Skill System
   ↓
Assessment
   ↓
Evaluation
   ↓
Insights
   ↓
Growth
   ↓
Hiring
```

This represents the **complete lifecycle of a learner**:

**Join → Learn → Test → Evaluate → Understand → Improve → Get Hired**
