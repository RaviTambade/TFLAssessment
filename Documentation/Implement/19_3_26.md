
# Building Employability in AI world

For almost **two weeks**, we have been discussing architecture, flow, and system design. Some of you may feel that *we haven’t written a single line of code yet*.

But there is a reason behind this.

Today, with AI tools and cloud development environments, we can generate **months of code within minutes**. A tool like **GitHub Copilot** or **ChatGPT** can produce the same amount of code that earlier took developers **six or seven months** to write.

However, the real challenge today is **not writing code**.

The real challenge is **understanding the flow, the architecture, and the linkages inside the generated code**.

AI can generate the code quickly, but **understanding that code may still take months**.

That is why we are spending time understanding:

* System architecture
* Folder structures
* API flows
* Data models
* Layered design

Because if your mindset is limited to **working with only two or three files**, you will struggle when you face **AI-generated projects with hundreds of interconnected files**.

### Why We Changed Our Training Approach

This change did not happen suddenly.

Last year I was in Mumbai conducting sessions for **Centre for Development of Advanced Computing** (CDAC). I was teaching **C and C++**, and I noticed something important.

Many students had been admitted to the program, but **their fundamentals were weak**.
Some of them struggled with very basic things like:

* Declaring variables
* Understanding program flow
* Writing simple logic

At the same time, the **course deadlines and expectations were extremely high**.

So there was a mismatch between:

1. **Prerequisites (weak fundamentals)**
2. **Deadlines and expectations**
3. **Actual industry demands**

Because of this mismatch, students were struggling.

That experience made me rethink our approach.

### Our New Learning Model

After returning from that experience, I made a decision:

We will **not follow a traditional classroom model anymore**.

Instead, we will run our program the way **industry actually works**.

This does **not mean we are ignoring fundamentals**.

Instead, we are **strengthening fundamentals by observing them inside real systems and real codebases**.

### AI Tools Are Part of Our Learning

* **GitHub Copilot**

That learning is far more valuable than **just typing the code manually**.


### How I Teach Now

You may have noticed that I also changed the way I conduct sessions.

While we are using AI tools extensively, there is **one thing you must always remember**.

Do **not become dependent on AI**.

Use AI tools for:

* learning
* exploring
* improving productivity

But always keep your **own thinking active**.

If you stop thinking and rely completely on AI, then your growth will stop.

# Mentor Guidance – Database, SQL Thinking, and the AI Era

## 1️⃣ Database First – Think in Data

Before writing application code, we must understand **data and database structure**.

Right now our focus is:

* Designing the **database schema**
* Creating a **`.sql` file**
* Writing **CREATE TABLE statements**
* Preparing **sample input data**

Example structure:

```sql
CREATE TABLE Users (
    UserId INT PRIMARY KEY,
    Name VARCHAR(100),
    Email VARCHAR(100),
    Role VARCHAR(50)
);
```

Every student should eventually have a **database script like this**:

```text
database_schema.sql
```

Inside it:

```text
CREATE TABLE ...
CREATE TABLE ...
CREATE TABLE ...
```

These scripts should represent the **entire database of the system**.


## 2️⃣ Identify Master Tables

Before writing sample data, we must identify **master tables**.

Examples of master tables:

```text

Roles

```

Three weeks ago many of you already generated **sample data** using AI tools.
