# 🔄 End-to-End Request Flow

```
User opens Web App
        ↓
Frontend sends request
        ↓
API Gateway authenticates request
        ↓
Request routed to correct microservice
        ↓
Service processes business logic
        ↓
Database queried
        ↓
Response returned
        ↓
Frontend updates UI
```

# 🧠 Why This Architecture Matters

This diagram helps students understand:

### 1️⃣ Systems are Layered

```
UI
↓
API
↓
Services
↓
Data
```

### 2️⃣ Services Are Independent

Each service solves **one business capability**.

Example:

```
User Service → manages users
Assessment Service → evaluates tests
Skill Service → maintains skill graph
```

### 3️⃣ Events Enable Scalability

When something happens:

```
Assessment Completed
        ↓
Event Published
        ↓
Other services react
```

This is why messaging systems like **RabbitMQ** are widely used.


### 4️⃣ Data Drives Everything

Every software system ultimately revolves around **data**.

Databases like **MySQL** store the core information that powers the application.
