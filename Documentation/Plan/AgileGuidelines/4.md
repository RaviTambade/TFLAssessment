
# 🧩 Full-Stack User Story Flow 

Example User Story:

**“Student wants to view assessment results.”**

```
+---------------------------------------------------+
|                   USER (Student)                  |
|         Opens browser and clicks "View Result"    |
+---------------------------+-----------------------+
                            |
                            v
+---------------------------------------------------+
|                FRONTEND (UI Layer)                |
|  HTML / CSS / JavaScript / React / Angular        |
|                                                   |
|  - Renders Result Page                            |
|  - Sends HTTP Request to Backend API              |
+---------------------------+-----------------------+
                            |
                            |  HTTP Request
                            v
+---------------------------------------------------+
|                  API LAYER                        |
|            REST API / Web API Controller          |
|                                                   |
|  Example Endpoint:                                |
|  GET /api/students/{id}/assessment-results        |
+---------------------------+-----------------------+
                            |
                            v
+---------------------------------------------------+
|             BUSINESS LOGIC / SERVICE LAYER        |
|                                                   |
|  - Validate Request                               |
|  - Apply Business Rules                           |
|  - Calculate Score                                |
|  - Prepare Response                               |
+---------------------------+-----------------------+
                            |
                            v
+---------------------------------------------------+
|                DATA ACCESS LAYER                  |
|                                                   |
|  - Repository Pattern / ORM (EF Core / JPA)       |
|  - Query Database                                 |
+---------------------------+-----------------------+
                            |
                            v
+---------------------------------------------------+
|                     DATABASE                      |
|                                                   |
| Tables:                                           |
|  Students                                         |
|  Assessments                                      |
|  Questions                                        |
|  StudentResponses                                 |
|  Results                                          |
+---------------------------+-----------------------+
                            |
                            |
                            v
+---------------------------------------------------+
|              RESPONSE FLOW BACK TO USER           |
|                                                   |
|  Database → Service → API → Frontend UI           |
|                                                   |
|  Student sees:                                    |
|  Score | Correct Answers | Attempted Questions    |
+---------------------------------------------------+
```

# 🔄 End-to-End Request Flow

You can also show them the **simplified system pipeline**:

```
USER ACTION
    |
    v
Frontend UI
    |
    v
REST API
    |
    v
Business Logic
    |
    v
Database
    |
    v
Response
    |
    v
Frontend UI Update
```

# 🧠 What Mentees Should Learn From This

Every **User Story touches multiple layers**:

```
User Story
     |
     v
UI Change
     |
     v
API Endpoint
     |
     v
Business Logic
     |
     v
Database Query
     |
     v
Working Feature
```

This is why companies hire **Full-Stack Developers**.

Not just people who know **syntax**, but people who understand **how systems work end-to-end**.


# 🎯 Mentor Tip for Your Session

When assigning a user story, ask students:

```
1️⃣ What UI change is needed?
2️⃣ What API will support it?
3️⃣ What business logic is required?
4️⃣ What database tables are involved?
```

If they answer these **four questions**, they are already thinking like **software engineers**.
