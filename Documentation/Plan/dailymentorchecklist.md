# 🎯 TFL CoMentor – Daily Mentor Review Checklist


## 🟢 1️⃣ Code Quality Review

#### ✅ Architecture

* [ ] Correct folder structure
* [ ] Clear separation of concerns
* [ ] No business logic inside controller
* [ ] No DB logic inside UI layer

#### ✅ Clean Code

* [ ] Meaningful variable names
* [ ] Small methods (< 30 lines ideal)
* [ ] No duplicate code
* [ ] Proper error handling
* [ ] No commented-out dead code

#### ✅ Standards

* [ ] Follows language conventions
* [ ] Proper indentation
* [ ] No hardcoded secrets
* [ ] Uses configuration properly

 

## 🔵 2️⃣ Technical Depth Check

Ask verbally:

* Can the student explain why this design?
* What happens if traffic increases?
* Where is bottleneck?
* What are edge cases?
* What is time complexity (if applicable)?

Score:

| Level | Observation            |
| ----- | ---------------------- |
| 1     | Copy-paste coding      |
| 2     | Understands syntax     |
| 3     | Understands logic      |
| 4     | Understands design     |
| 5     | Thinks architecturally |

 

## 🟣 3️⃣ Testing Discipline

* [ ] Unit tests written?
* [ ] Edge cases covered?
* [ ] Negative scenarios tested?
* [ ] Coverage > target threshold?
* [ ] Tests fail when logic breaks?

Ask:

> “If I change this logic, will your test detect it?”

 

## 🟠 4️⃣ DevOps Discipline

* [ ] Proper Git commits (clear message)
* [ ] Feature branch used
* [ ] Pull request created
* [ ] CI pipeline passing
* [ ] Docker build successful

Check commit message quality:
❌ “update code”
✅ “Add RBAC middleware for admin routes”

 

# 🔴 5️⃣ Debugging Ability

Instead of solving:

Observe:

* [ ] Student reads logs
* [ ] Student isolates issue
* [ ] Student reproduces error
* [ ] Student tries multiple approaches
* [ ] Student documents root cause

Score persistence level (1–5).

 

## 🟡 6️⃣ Communication & Ownership

* [ ] Can explain flow without hesitation
* [ ] Accepts feedback without ego
* [ ] Asks intelligent questions
* [ ] Takes ownership of bugs
* [ ] Updates documentation

---

## 🧠 7️⃣ Growth Mindset Indicators

Look for:

* Curiosity beyond assignment
* Refactoring without being told
* Suggesting improvements
* Exploring performance metrics
* Reading official documentation

## 🏢 10-Minute Fast Review Version (When Time Is Limited)

Ask 5 questions:

1. Show me your commit.
2. Show me your test coverage.
3. Show me logs.
4. Show me API response.
5. Explain one design decision.

If they cannot answer confidently → revisit fundamentals.

 
### 🧠 Powerful Mentor Rule

Never say:

> “This is wrong.”

Instead ask:

> “What happens if 1 lakh users hit this endpoint?”

Make them discover flaws.

 

## 📅 Weekly Meta-Review (Every Friday)

Add:

* Improvement since Monday?
* Repeated mistakes?
* Confidence level change?
* Technical clarity growth?
* Risk of stagnation?
